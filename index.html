A hands on introduction to the building and hosting websites from scratch

Have you ever wanted to create and host your own website? Starting from scratch can be time consuming and hard to learn, even if you already know a little HTML and CSS. Create an index.html file

What is a static web page?

Most websites you visit live on a server, a fancy word for a computer accessible from the public internet. But there’s another kind of website – a static website – that is much more basic.

Static websites don’t need to be run (or hosted) on servers like traditional websites. All their code runs in the browser. Since they don’t talk to databases, they can be stored in what is called object storage. People can access these sites whenever they are needed and don’t always need to be running, which makes them significantly cheaper than hosting a website on a server.

Why would I want a static website?

Static websites can’t communicate directly with a database but can retrieve and send data over API calls. In fact these days, most browsers are so powerful there isn’t a lot they can’t do. They also offer many benefits that include:

  • Low cost: static websites are significantly cheaper than server-based websites, even one hosted by CMSs like WordPress
  • High scaling: static websites are so simple they scale effortlessly, even at high volumes of traffic
  • Easy deployments: anyone who can create a basic HTML document can deploy one in a few minutes

Writing your own HTML

When you visit a webpage, you are really visiting a file (or group of files) that’s stored on a different computer. Special software called a server makes that file available to the public. These files contain everything needed to display the page – text, styles, images and code to load other resources.

When your browser (like Chrome, Firefox or Edge) displays a web page, it translates these files to show you the text, styling and actions embedded in that code.

Web developers use special programs to create and editor their code, but HTML files are just text documents. They can be created in any text editor, but some add additional formatting that can interfere with the code. Simpler text editors are usually better.

Basic HTML tags

You can think of tags like a container that enclosing everything ‘inside’ the tag with a specific meaning or style. The only things that can go inside the <html></html> tags are plain text or other HTML tags – but you have plenty of tags to choose from.

There are two kinds of HTML tags you can use:

  • Semantic Tags like <main></main>, <section></section>, heading and list tags. Semantic tags come with additional built-in features and are important landmarks for people using accessibility devices.
  • Non-semantic Tags like <div> and <span> do not carry additional meaning and are good for stylistic elements and containers

Styling content

All browsers have default (or fallback) styles they use to display a webpage when no specific styles are provided. If you’re looking for more impact, you can add (almost) any style you imagine to HTML elements. Styles can be added directly to elements, but more often they will be added by writing another language called CSS inside a pair of <style></style> tags or through a seperate .css file linked in the <head></head>.

Host your static site on GitHub

What is GitHub?

GitHub is a website where you can share and collaborate on code. It has many advanced features, but it’s also one of the easiest ways to host a static website using a feature called GitHub Pages. Best of all, it’s completely free!

GitHub is based on a language called Git, which stores changes (or commits) in different branches. It can compare, combine, and keep track of those changes. Git is very powerful – learning even the basic commands makes development significantly easier.

Sign up for a GitHub Account

Signing up for a GitHub account is simple. Just go to their signup page and provide and email and a password to create a new account.

Create a GitHub repository

All code in GitHub lives inside a GitHub Repository (often called a repo).

Creating a repo is easy:

  1. Click on the Repositories tab on the top navigation bar.
  2. On the next page, you should be prompted to create your first repository. If not, click the green New button to the right of the search bar.
  3. On the repository setup screen, you only need to make a few changes:
    1. Give a name to your project. This name will be used to access your site, so pick something memorable!
    2. Make sure the repository is marked as Public
    3. Select the checkbox that says Add a Readme File
    4. Click the green Create Repository button

    4. Once you've finished creating your new repository, you should be taken to its homepage automatically.

Your repository’s README.md

Your new repository will be empty except for the README.md you just asked the setup wizard to create. Readme.md is written in another language called markdown.

You can think of markdown as a simplified version of HTML, but you can still do a lot with it. The README.md file is intended to provide information about a repo, but people use them for plenty of creative purposes!

Add index.html to your repository

There are many ways to add code to GitHub, but the easiest ways are directly through the website.

  1. Option A: Click Add File then Upload Files and select the .html file you created earlier Option B: Click Add File then Upload Files and select the .html file you created earlier
  2. Click the green Commit changes button on the upper right
  3. A dialog box will pop up asking for more details about the changes you’re making (the commit). Just click Commit changes a second time to close this dialog
  4. Your index.html file should be visible inside the repository.

What is GitHub Pages?

If you click on index.html, you can all see the code you wrote. But the browser isn’t interpreting it as a web page -- the code just looks like text.

For the browser to interpret this HTML, you need to configure the repository for public hosting. GitHub has a feature called GitHub Pages you can use to host simple pages like this for free.

Configure GitHub Pages to host index.html

You can use another GitHub feature called GitHub actions to deploy your site automatically every time you change the code in the repository.

  1. Click the Settings tab on top of the repository
  2. Select Pages from the navigation bar on the right
  3. Under Build and Deployment heading, make sure Source is set to GitHub Actions
  4. In the Static HTML panel that appears, click Configure
  5. Some code will appear, but you don’t need to make any change. Just click Commit Changes on this screen and the next dialog that appears

Navigate back to the Pages section of settings. Now you should see a banner at the top of the page with the link to your live website.

The default address of this website makes it clear this site is hosted on GitHub, but you can configure a custom domain name. In the next section you will learn another way of hosting static websites with more advanced features: hosting directly in the cloud.

Create an AWS Account

What is the Cloud?

Put simply, the Cloud is a bunch of different computers around the world that work together to perform the same tasks. This lets us share content faster, process data more efficiently, and save costs by sharing computing resources.

There are several different Clouds which all offer a similar functionality. I’ve chosen to use Amazon Web Services (AWS) in this tutorial. Similar concepts should apply to any Cloud Provider.

The Cloud can have many benefits, but for simple websites the main value is cost savings. All Cloud Providers work on a “pay-as-you-go” model, meaning you will only pay for resources you actually use.

What is the AWS Free Tier?

AWS provides many services free to new users for the first twelve months. You will be charged for each service separately, so it’s important to check the billing page for each service you are using. AWS provides a calculator that can be helpful to get a rough idea of the costs of using multiple services together.

Sign up for an AWS Free Tier Account

Signing up for an AWS Account is a little more complex than GitHub and requires credit card information. Be careful – if you go over the limits of the Free Tier, AWS will charge you automatically.

  1. Go to AWS Free Tier and click on the yellow Create a Free Account button in the center of the page.
  2. On the next page, enter your email address and the name you’d like to use for your account.
  3. AWS will confirm you have access to this email address by sending you a verification code; enter this number and clikc Verify email address
  4. Complete the rest of the sign-up form then click Create account

(Optional) Create an IAM Role

The account you just created is called a root account – an account with all possible permissions. You can log in and work with this account, but it’s more secure to use a special, limited purpose role linked to your account called an Identity and Access Management (IAM) role. If you do create one, just make sure to give it permission to view the Billing and Cost Management Consoleso you can keep an eye on your spending.

(Optional) Set up alerts using AWS Budgets

It’s always a good idea to set up alerts for your AWS account in case you get an unexpected surge in traffic or experiment with a service and forget to turn it off. You can create an AWS Budget and set up budget alerts that will automatically let you know (by email or by text) when you’ve gone over your preset budget limit.

What are AWS Regions?

You may have noticed the region selector on the top right of the AWS Console.

Northern Virginia (us-east-1) is the default region, but all accounts can create resources in any AWS Region. You can change to one closer to you to decrease the response time from AWS Services. Some services are global and do not have regions. Other services like S3 are global, but you still need to pick a 'home' region for your bucket.

Host your static site in S3

What is S3?

You can think of an S3 Bucket like a file folder in the Cloud – but one that can be accessed by anyone with permission. This makes them great for hosting public content like images and static websites

Create an S3 Bucket

First, we'll create an S3 Bucket to hold your website.

  1. Search for S3 in the main search bar on the top of the page
  2. On the S3 Dashboard, click the orange Create bucket button on the far right
  3. On the next page, give your bucket a globally unique name. Think carefully – if you don’t buy a domain name, this is the name people will use to access the site.
  4. Under Block Public Access Settings uncheck the box marked All Public Access.
  5. A yellow warning will appear; click the checkbox to acknowledge AWS doesn’t recommend this (unless you are hosting a static website!) -- just make sure this Bucket won’t be used for anything else
  6. Scroll all the way to the bottom of the page and click the orange Create bucket button

Configure your S3 Bucket to host public content

Now that you’ve created a public S3 Bucket, there are just a few more steps to configure it for web hosting.

  1. On the S3 dashboard, select the Bucket you just created.
  2. Click the Properties tab at the top of the page, then scroll to the section called ‘Static website hosting’
  3. Click the grey edit button on the right of the Static website hosting’ section
  4. On the page that opens, click enable then fill in the name of your Index document (your home or default page). This is easy because we called it index.html!

Your bucket is now configured and ready to host your website! If you check the Static website hosting box again, you will see the updated information, including the default URL (Uniform Resource Locator) for your new site. If you visit it right now, you’ll get an error message -- the last step is adding your index.html file to the bucket.

Add content to your S3 Bucket

There are many ways to put your index.html file into S3 – including integrating it with GitHub! But the easiest way is to upload files manually using the S3 Dashboard.

  1. Click back to the Objects tab and click the orange Upload on the bucket to the right
  2. Select the index.html file you created earlier.
  3. Be patient -- it may take a few minutes before your website is available

(Optional) Add a custom domain name

What is a custom domain name?

You probably noticed the name for your website is a little hard to remember. That’s because S3 generate this name automatically by combining the name of the bucket with the region it was created in.

Everything on the internet, including your website, is identifiable by a string of unique numbers called an Internet Protocol (IP Address). IP Addresses make sending things to the right device possible -- but they aren’t particularly easy to remember or use, however.

Browsers let us find websites using their Domain Name -- something like amazon.com, mit.edu or whitehouse.gov. The Domain Name System (DNS) uses special servers called Nameservers to resolve Domain Names to their IP addresses and direct traffic to the correct site.

There are only so many of these names available for a given domain (the last part of the domain name after the period, like .com and .edu). There are only so many of these names available and you need permission to use certain domains. An organization called ICANN manages the ownership and expiration of Domain Names, but you don’t buy names through them directly. Instead, Domain Names are purchased through a registration service.

Buy a custom domain name on Route 53

There are many different domain registration services, but AWS also has its own service called Route 53. Route 53 lets you browse and buy new domain names, as well as configure domains names from different services.

  1. Search for Route 53 on the search bar on the top of the page
  2. On the Route 53 Dashboard, select Register domain on the far right
  3. Search for a domain name. The results will show you a lot of options with different prices and different domains -- if you find one you like, AWS can charge it automatically to the credit card you have on file.

What are domain name records?

When you registered your domain, Route 53 automatically created a Hosted Zone. A Hosted Zone contains a series of instructions called Records which contain instructions on how to route traffic to your Domain Name. If you registered a domain name through another website, you need to set up this Hosted Zone yourself.

You can create different records in this domain to tell Route 53 exactly how to route traffic looking for your website.

Configure your custom domain name

We’ll start by creating an A Record which points your new domain name to your website in S3.

  1. On the Route 53 Dashboard, select Hosted Zones from the sidebar on the left
  2. Select the Hosted Zone for your new custom domain name
  3. Click the orange Create record button
  4. On the next screen, click the Switch to wizard button
  5. Make sure Simple routing is selected and click Next
  6. Click Define simple record
  7. On the next screen, leave the subdomain value blank and make sure the record type is set to A – Routes Traffic to an IPv4 Address and some AWS Resources
  8. In the dropdown menu under Value/Route traffic to select Alias to S3 Website Endpoint
  9. Select the region where you created your S3 Bucket, then the URL for your website from the dropdowns below
  10. Click Create record. If you go back and look at your hosted zone, you should see that Route 53 has created two new records (NS and SOA records) automatically.

Be patient! It may take some time before your new Domain Name routes correctly to your site. These changes need to be made (or propagated) on many different name servers all over the world which can sometimes take several hours.

What can you do next?

Congratulations! Now you know how HTML goes from being a text document to being live and visible on the internet. These are just the basics, but you have plenty of options to continue improving your site:

  • Want a prettier site? Learn more about CSS and embedding content
  • Want to create an interactive site and load content dynamically? Learn about the JavaScript, it’s simpler form jQuery and getting data from a public API
  • Want to add forms and get feedback from your site? AWS has many serverless services like API Gateway, Lamba and SNS that can help you
  • Want to go global? Use a Content Delivery Network (CDN) like Amazon CloudFront to serve your content anywhere at lightning-fast speeds.
  • See a warning from your browser that your site isn’t secure? Request an authentication certificate through Amazon Certificate Manager to deliver your site over HTTPS instead of HTTP
  • Ready to learn about more advanced websites? Find out more about setting up a server, creating databases, or dive into one of the many web development frameworks used by larger sites.
